Rewrite clustering.py with the following logic:

1. When a session is uploaded, check its cluster_id and generated_on (ASUP capture time)
2. Find an EXISTING group within the same cluster whose capture times are within ±20 minutes of this session's capture time
3. If found → add this session to that group
4. If not found → create a NEW group and add this session
5. NO limit on group member count — a group can have 2, 10, or any number of sessions
6. Sessions with capture times outside the 20-min window get a new group

Implementation approach:
- Query existing groups for this cluster_id (via SessionGroup + SessionGroupMember)
- For each group, check the min/max generated_on of its members
- If new session falls within [min-20min, max+20min] of any group → join it
- Otherwise create new group

Database models:
- SessionGroup: id, cluster_id, created_at
- SessionGroupMember: group_id, session_id
