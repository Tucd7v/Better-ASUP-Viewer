Fix HA pairing failing on slow networks: session data loads asynchronously and useMemo on `sessionRows` doesn't always re-trigger when partnerHostname arrives after initial render.

Problem: partnerHostname arrives after sessions are already rendered. The haPairSessionIds memo runs with stale/empty partner data and doesn't recompute when partnerHostname fills in later.

Fix: Add a `partnerHostnameKey` computed from sessionRows' partnerHostname values. When this key changes (new partner data arrives), haPairSessionIds recomputes.

```tsx
const sessionRows = ... // existing
const partnerHostnameKey = sessionRows.map(s => s.partnerHostname ?? '').join(',')

const haPairSessionIds = useMemo(() => {
  // if partnerHostnameKey is empty (all empty), skip computation
  if (!partnerHostnameKey) return new Set<string>()
  // ... existing computation ...
}, [sessionRows, partnerHostnameKey])  // <-- add partnerHostnameKey
```
