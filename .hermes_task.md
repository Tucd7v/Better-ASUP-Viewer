Investigate and fix: ColSwapMenu popup position is correct in Grid mode but wrong on first open in Canvas mode.

## Background
In `openSwapMenu()`, position is calculated from:
```
thRect = th.getBoundingClientRect()  // column header position
wRect = wrapperRef.current.getBoundingClientRect()  // card outer div position
left = thRect.left - wRect.left
top = thRect.bottom - wRect.top + 4
```

## Hypothesis
In Canvas mode, the card renders inside a ReactFlow node wrapper. On first render:
1. The card's content (table rows) loads asynchronously via `getFileContent()`
2. `useResizable` initially returns a small height, then updates when content loads
3. The `wrapperRef`'s `getBoundingClientRect()` might return incorrect height/position before content is fully rendered
4. In Grid mode, the card is in a fixed-size CSS Grid cell, so dimensions are stable

## Task
1. Confirm the root cause by adding console.log in `openSwapMenu` to log thRect, wRect, and calculated position
2. Fix: wrap the measurement in `requestAnimationFrame` OR defer it until `rows.length > 0`
3. Test: open XML file in canvas mode → click ⇄ → popup should appear below the column header
4. Also test in Grid mode to ensure it still works
5. Remove debug logs after fixing
