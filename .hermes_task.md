# Task: Fix analysis mode context_file_ids bugs (2 issues)

## Bug 1: AI doesn't know which files to read
Change `user_message` generation in BOTH `/chat` AND `/chat/stream` endpoints.
The `user_message` for context mode currently says:
```
"查看用户画布内容（已打开文件分析模式）\n\n用户问题：..."
```
But it needs to include actual file IDs and names. Change to:
```python
    if body.context_file_ids:
        # Build brief file list for the AI from the catalog
        file_list_str = ""
        for entry in catalog:
            if entry["file_id"] in body.context_file_ids:
                file_list_str += f"  [{entry['file_id']}] {entry['filename']} ({entry['file_type']})\n"
        if file_list_str:
            user_message = f"查看用户画布内容——以下文件已打开，请仅分析这些文件：\n\n{file_list_str}\n\n用户问题：{body.message}"
        else:
            user_message = f"查看用户画布内容（已打开文件分析模式）\n\n用户问题：{body.message}"
```

IMPORTANT: The `catalog` is returned from `_build_context()`. Both endpoints currently only destructure `_, execute_tool, session_info, catalog_by_session`. Need to also get `catalog`:

```python
    catalog, execute_tool, session_info, catalog_by_session = await _build_context(...)
```

## Bug 2: Empty context_file_ids bypasses restrictions
In `execute_tool`, change:
```python
        if context_file_ids:
```
to:
```python
        if context_file_ids is not None:
```

Python: empty list `[]` is falsy, so `if []:` skips the block. `is not None` correctly detects "context mode is active."
