# Task: Add SplitMode to NodeASUP Viewer

## Goal
Add a "SplitMode" toggle button to the template bar. When active, the ReactFlow canvas is replaced by a CSS Grid layout with 1-4 fixed zones. Users drag files from the left FileTree directly into these zones. Card sizes adapt to browser window size in real-time.

## Files to Modify


**Add toggle button in template bar** (next to Clear button):

```tsx
<button onClick={() => setSplitMode(!splitMode)} style={{...}}>

</button>
```

**Add split layout render when splitMode is true:**

When splitMode is active, do NOT render the ReactFlow canvas. Instead render a CSS Grid container with zones. The grid layout depends on how many cards are currently on the canvas:

- 1 card: 1 column (full width)
- 2 cards: 2 columns (equal split)
- 3 cards: 3 columns (equal split)
- 4 cards: 2x2 grid

```tsx
// The grid occupies the same space as ReactFlow (main.viewer-main)
const gridCols = cardCount <= 1 ? 1 : cardCount <= 3 ? cardCount : 2
const gridRows = cardCount <= 3 ? 1 : 2

<div style={{
  display: 'grid',
  gridTemplateColumns: `repeat(${gridCols}, 1fr)`,
  gridTemplateRows: `repeat(${gridRows}, 1fr)`,
  width: '100%', height: '100%',
  gap: 8, padding: 8,
}}>
  {visibleCards.map((node, idx) => {
    const CardComponent = nodeTypes[node.type!]
    return (
      <div key={node.id}
        data-zone={idx}
        style={{ border: '2px dashed #e2e8f0', borderRadius: 8, overflow: 'auto', position: 'relative' }}
        onDragOver={(e) => e.preventDefault()}
        onDrop={(e) => {
          e.preventDefault()
          const fileId = e.dataTransfer.getData('text/plain')
          if (fileId) handleFocusFile(fileId)
        }}
      >
        {CardComponent && <CardComponent data={node.data} />}
      </div>
    )
  })}
  {/* Empty drop zones for remaining slots */}
  {Array.from({ length: Math.max(0, 4 - cardCount) }).map((_, i) => (
    <div key={`empty-${i}`}
      style={{ border: '2px dashed #cbd5e1', borderRadius: 8, display: 'flex', alignItems: 'center', justifyContent: 'center', color: '#94a3b8', fontSize: 13 }}
      onDragOver={(e) => e.preventDefault()}
      onDrop={(e) => {
        e.preventDefault()
        const fileId = e.dataTransfer.getData('text/plain')
        if (fileId) handleFocusFile(fileId)
      }}
    >
      Drop file here
    </div>
  ))}
</div>
```

**Important: The existing handleFocusFile already handles adding new cards.** When a file is dropped, the fileId is extracted from `e.dataTransfer.getData('text/plain')` and passed to handleFocusFile. This adds the card to the nodes array, which triggers a re-render with the new grid layout.

**Resize handling:** The grid uses CSS `fr` units which are inherently responsive to container size. No extra ResizeObserver needed. The `viewer-main` div already fills the available space (flex: 1).

## Constraints

- **splitMode OFF**: everything works exactly as before (ReactFlow canvas)
- **splitMode ON**: ReactFlow canvas is hidden, grid layout is shown
- **Max 4 cards**: if user tries to add a 5th card, it replaces the card in the last slot
- **Drag from FileTree**: should still work in normal mode (click to add)
- **Template bar**: remains visible and functional in both modes
- **AI panel**: should still be accessible in split mode
- **Card component rendering**: use the existing `nodeTypes` mapping and `<CardComponent data={node.data} />` pattern. The card components are: TextFileCard, XMLFileCard, EMSFileCard (defined at top of ViewerPage.tsx)
- **Node data structure**: each node has `data: { fileId, sessionId, filename, nodeColor, collapsed, onCollapse, onHide }`

## Key Implementation Details

1. **hide ReactFlow when splitMode=true**: wrap ReactFlow in `{!splitMode && (<ReactFlow...>)}`
2. **show grid when splitMode=true**: `{splitMode && (<div className="viewer-main" style={...}>...grid...)}`
3. **card count**: `const cardCount = nodes.filter(n => !state.hiddenFileIds.has(n.data.fileId)).length`
4. **maximum 4 limit**: in handleFocusFile, when splitMode is true, if nodes already have 4 visible cards, replace the last one instead of adding
5. **Toggle button style**: use the same style as other template bar buttons (`templateBtnStyle` constant in ViewerPage.tsx — gray outline, small font)
