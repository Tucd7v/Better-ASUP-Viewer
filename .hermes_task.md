Fix XML card centering on spawn. Currently TXT/EMS cards center correctly when created, but XML cards drift off-center after their content loads.

## Root cause
In ViewerPage.tsx `handleFocusFile` (around line 720), all cards use `CARD_W = 340` for initial positioning. After XML cards load their data, they call `setWidth()` which changes the rendered width, but ReactFlow's `fitView` was already called based on the old dimensions.

From the card defaults:
- TextFileCard: useResizable(900, 340)
- EMSFileCard: useResizable(800, 400)  
- XMLFileCard: useResizable(320, 360) — then overrides width at runtime via setWidth()

## Requirements
1. Clean way to center ANY card type on spawn, regardless of dynamic resizing
2. No viewport jitter when creating multiple cards quickly
3. No hardcoded delays (setTimeout) — use proper lifecycle hooks

## Suggestions
- Use `onNodesChange` or `useOnViewportChange` from ReactFlow to detect when nodes are added
- Or expose a callback from each card component that fires when content is loaded and dimensions are stable
- Or use ResizeObserver on the ReactFlow pane to detect when node dimensions change
- Choose the approach that gives the smoothest result
