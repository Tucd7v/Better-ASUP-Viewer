Fix the drag conflict between Grid card grip (⋮⋮) and XML table column reorder.

## Problem
When dragging a card's grip in Grid mode, the browser's native HTML5 drag system triggers `onDragEnter` on XML `<th>` elements (which are also `draggable` for column reordering). Both drag systems activate simultaneously.

## Fix
Add a module-level flag that the grip sets when starting a card drag, and XML's column drag handlers check before executing.

### In TextFileCard.tsx / EMSFileCard.tsx / XMLFileCard.tsx
Modify GridDragGrip:
- In onDragStart: set a module-level `isGridDragActive = true` 
- In onDragEnd: set `isGridDragActive = false`

### In XMLFileCard.tsx
In the column reorder handlers (onDragStart, onDragEnter, onDragEnd at ~lines 312-330):
- At the start of each handler: `if (isGridDragActive) return`

### Implementation detail
Use `let isGridDragActive = false` as module-level variable (NOT React state — it must be synchronous and outside the render cycle).

The flag should be exported or shared between card files. Best approach: define it in `ViewerPage.tsx` since that's where SplitGrid/SplitCard live, and the grip is triggered from there. But the xml column handlers are in XMLFileCard...

Simpler: define `window.__gridDragActive` on the window object, check it in XMLFileCard.
Or: define the flag in `useResizable.ts` or a new shared module.

Actually the simplest: export `isGridDragActive` from XMLFileCard and import it in the other two files. But three files all checking one flag.

Cleanest approach: create a tiny shared module `src/features/viewer/nodes/gridDragState.ts`:
```ts
let active = false
export function setGridDragActive(v: boolean) { active = v }
export function isGridDragActive() { return active }
```

All three card files import from it. SplitCard doesn't need to know about it — the grip handles it locally.
