Fix: Analysis mode XML file reading broken

In AIChatPanel.tsx, getVisibleFileIds() currently uses state.fileList (global ViewerContext - ALL files from ALL sessions). This means analysis mode sends wrong or incomplete context_file_ids, especially after multi-tab changes.

Fix: Pass the active tab's canvas node file IDs as `tabFileIds` prop to AIChatPanel. Then getVisibleFileIds should use tabFileIds instead of state.fileList.

In AIChatPanel:
- Add `tabFileIds?: string[]` prop
- Change getVisibleFileIds to return tabFileIds if provided, otherwise fall back to state.fileList

In ViewerPage:
- Derive tabFileIds from activeTab.nodes (visible cards on canvas)  
- Pass to AIChatPanel

Files: frontend/src/features/viewer/AIChatPanel.tsx, frontend/src/features/viewer/ViewerPage.tsx
