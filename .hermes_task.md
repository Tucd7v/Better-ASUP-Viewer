# Fix SplitMode: card height + drag behavior

## Issue 1: Cards don't fill vertical space
The content area inside each card component uses `height` from `useResizable()` hook, which is a fixed pixel value. In SplitMode, the inner content div should use `flex: 1` and `overflow: auto` to fill remaining space within the flex-column container.

Files to fix:
- `frontend/src/features/viewer/nodes/TextFileCard.tsx` — the `<div ref={listRef}>` at ~line 159 has `style={{ height, overflowY: 'auto', ... }}`. In splitMode, change `height` to `undefined` and add `flex: 1`.
- `frontend/src/features/viewer/nodes/EMSFileCard.tsx` — find the events list container and apply same fix.
- `frontend/src/features/viewer/nodes/XMLFileCard.tsx` — find the table container (the div with `overflowX: 'auto'`) and apply same fix.

For each card's content area div:
```tsx
style={{ 
  height: splitMode ? undefined : height, 
  overflowY: 'auto', 
  flex: splitMode ? 1 : undefined,
  ...
}}
```
