# Task: Fix auto-detect mode — sync with canvas state in real-time

## Problem
useState initializer only runs on first mount. If user opens AI panel before opening cards, mode stays 'autonomous' even after cards appear. React may also preserve component across showAI toggles.

## Fix in frontend/src/features/viewer/AIChatPanel.tsx

Replace lines 26-27 (the useState for mode):

```tsx
  // Track canvas state for auto-detection
  const visibleCount = state.fileList.filter(f => !state.hiddenFileIds.has(f.id)).length
  const [mode, setMode] = useState<'analysis' | 'autonomous'>('autonomous')
  const [userToggledMode, setUserToggledMode] = useState(false)

  // Sync mode with canvas state — unless user manually toggled
  useEffect(() => {
    if (!userToggledMode) {
      setMode(visibleCount > 0 ? 'analysis' : 'autonomous')
    }
  }, [visibleCount, userToggledMode])
```

Then update BOTH toggle buttons to set userToggledMode:

  分析模式 button onClick: `() => { setMode('analysis'); setUserToggledMode(true) }`
  自主模式 button onClick: `() => { setMode('autonomous'); setUserToggledMode(true) }`

DO NOT change anything else. Build and commit.
