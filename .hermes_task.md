Fix two XMLFileCard issues in frontend/src/features/viewer/nodes/XMLFileCard.tsx:

## Issue 1: Title + hostname not vertically centered
The filename text and hostname text don't align with the icon (📊) and buttons ([−] [×]) in the header row.

The parent flex container uses `alignItems: 'center'` but the inner text elements have different vertical positions. The filename has `fontSize: 12` (implicit lineHeight ~14px) while the hostname has `fontSize: 11`.

Fix: Add explicit `lineHeight` to both text elements so they share the same baseline. Try `lineHeight: '16px'` on both the filename span and hostname span. Or try `alignSelf: 'center'` on the headerTitle wrapper.

## Issue 2: ColSwapMenu opens at wrong position on first render
When first opening an XML card, clicking ⇄ shows the popup at an incorrect position. After toggling Grid mode on/off, it corrects itself.

This is because `getBoundingClientRect()` reads dimensions before the card's content has finished layout. The table rows load asynchronously (fetch content), so the th elements might be at wrong positions on first render.

Fixes to try:
1. Wrap the measurement in `requestAnimationFrame(() => { ... })` 
2. OR: use a `useEffect` that tracks when rows are loaded, and only enable the swap menu after data is present
3. OR: add a `setTimeout(() => { ... }, 0)` after setAllColumns/setColumns to let React commit the DOM update before measuring

The best approach: use `requestAnimationFrame` inside `openSwapMenu`, but ALSO check that `rows.length > 0` before showing the menu.
