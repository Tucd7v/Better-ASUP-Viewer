Fix card state loss on swap in Grid mode.

## Current problem
`handleSwapCard` in ViewerInner swaps nodes in the parent array via `setNodesForTab`. This causes the parent to re-render, which causes SplitGrid to re-render with new `gridNodes` order. Even though React uses `key=node.id`, the child card components somehow lose internal state (scroll position, search/highlight, etc.).

## Fix: Local cardOrder in SplitGrid
SplitGrid maintains its own `cardOrder` state (array of node IDs). Swapping only changes `cardOrder` — parent `nodes` NEVER changes.

### Implementation in SplitGrid
```tsx
// Local order state
const gridNodeIds = gridNodes.map(n => n.id)
const [cardOrder, setCardOrder] = useState<string[]>(gridNodeIds)

// Sync cardOrder when gridNodes change (new cards added/removed)
const nodeIdKey = gridNodeIds.join(',')
useEffect(() => {
  setCardOrder(prev => {
    // Keep existing order, append new IDs
    const existing = new Set(prev)
    const retained = prev.filter(id => gridNodeIds.includes(id))
    const added = gridNodeIds.filter(id => !existing.has(id))
    return [...retained, ...added]
  })
}, [nodeIdKey])

// Render in cardOrder sequence
const orderedNodes = useMemo(() => {
  const map = new Map(gridNodes.map(n => [n.id, n]))
  return cardOrder.map(id => map.get(id)).filter(Boolean) as Node[]
}, [cardOrder, gridNodes])

// Swap handler
const handleSwap = (src: number, tgt: number) => {
  setCardOrder(prev => {
    const next = [...prev]
    [next[src], next[tgt]] = [next[tgt], next[src]]
    return next
  })
}
```

### Remove handleSwapCard from ViewerInner
- Delete `handleSwapCard` useCallback
- Remove `onSwapCard` from SplitGrid props
- Remove `onSwapCard` from SplitGrid call site

### key props on SplitCard
Important: every SplitCard must have `key={node.id}` to prevent unmount/remount. Already done.
