Feature: clicking the hostname in card headers scrolls the FileTree sidebar to focus on that node.

## Flow
1. User clicks hostname label in card header (TextFileCard / EMSFileCard / XMLFileCard)
2. Dispatches `viewDispatch({ type: 'FOCUS_NODE', hostname: 'node01A' })` 
3. FileTree listens to viewer context for `focusNode` state
4. FileTree scrolls to the matching node, briefly highlights it (flash CSS class)
5. Timer removes highlight after 2 seconds

## Implementation

### ViewerContext (ViewerContext.tsx)
Add to state: `focusNode: string | null`
Add action type: `'FOCUS_NODE'` with payload `{ hostname: string }`
Add action: `'CLEAR_FOCUS_NODE'` 

### Card components (TextFileCard.tsx, EMSFileCard.tsx, XMLFileCard.tsx)
The hostname text is currently a `<span>`. Change it to a `<button>` with `cursor: pointer` and `onClick` that dispatches `FOCUS_NODE`. Keep styling same (color #94a3b8, fontSize 11). Add underline on hover.

### FileTree (FileTree.tsx)
Add useEffect watching `state.focusNode`. When set:
- Find the session/div that matches the hostname
- `scrollIntoView({ block: 'center', behavior: 'smooth' })`
- Add flash class for 2s then dispatch `CLEAR_FOCUS_NODE`

### CSS (index.css)
Add `.file-tree-node-flash` with brief background highlight animation.
