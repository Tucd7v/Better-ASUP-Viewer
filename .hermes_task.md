Three related changes for multi-node HA support:

## 1. NodeHUD: 4 nodes → 2×2 layout, pagination for overflow

Change grid logic:
- 4 nodes: use `grid-template-rows: repeat(2, auto)` (2 per column)
- 5-6 nodes: use `grid-template-rows: repeat(3, auto)` (3 per column)
- 7+ nodes: pagination — show 6 at a time with ← → buttons, `grid-template-rows: repeat(3, auto)`

```ts
const rowsPerCol = rows.length === 4 ? 2 : 3
const maxVisible = rowsPerCol * 2  // 2 columns max
// If rows.length > maxVisible, add page state
```

In `frontend/src/features/viewer/NodeHUD.tsx` and `frontend/src/index.css`.

## 2. Backend: parse storage-failover.xml for HA partner info

Add a new backend endpoint or extend existing session info to include partner hostname. When files are uploaded, check if `storage-failover.xml` exists among the uploaded files, parse it for the `Partner Name` column, and store the partner hostname in the session record.

In `backend/models/db.py`, add `partner_hostname` column to Session.
In `backend/services/asup_parser.py` or a new function, parse storage-failover.xml.
In the session info API response, include partner_hostname.

## 3. Frontend FileTree: HA pair visual connector

When two sessions have matching partner info (session A's partner = session B's hostname), draw a visual connector line between them in the FileTree sidebar.

- Add `partnerHostname` to SessionMeta type in `frontend/src/types/index.ts`
- Pass partner info through the API response chain
- In FileTree.tsx, when two sessions form an HA pair, render a vertical bracket/line connecting their rows

Simplest implementation: a CSS border-line on the left side connecting paired nodes.
```tsx
{isHAPair && <div style={{position:'absolute', left: 4, top: 0, bottom: 0, borderLeft: '2px solid #3b82f6'}} />}
```
