Templates should remember whether they were saved while in Grid (Split) mode and restore that mode when loaded.

## Changes needed

### 1. Add splitMode to TemplateCard type
In `frontend/src/types/index.ts`, add `splitMode?: boolean` to `TemplateCard`.
In `backend/schemas/api.py`, add `split_mode: Optional[bool] = False` to the TemplateCard schema.

### 2. Save splitMode when creating template
In `ViewerPage.tsx`, `handleSaveTemplate` — when creating a template, include the current `splitMode` state in the nodes/edges data for each card. Pass `split_mode` to the API.

### 3. Restore splitMode when loading template
In `ViewerPage.tsx`, `handleLoadTemplate` — when loading a template, check if any card has `splitMode: true` and if so, set the viewer to split mode (`setSplitMode(true)`).

### 4. Backend: store split_mode
In `backend/api/templates.py`, the template create/update endpoints should accept and store `split_mode` in the template JSON.

### Implementation plan (minimal)
The simplest approach: store `splitMode` as a top-level field on the template (not per-card). When saving, include `splitMode: currentSplitMode`. When loading, if template has `splitMode: true`, toggle to grid mode.
