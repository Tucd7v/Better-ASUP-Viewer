Add auto-analysis after upload: AI scans key ASUP files and generates a health summary.

## 1. New system prompt (services/llm.py)
Add POST_UPLOAD_SYSTEM_PROMPT — focused on automated health check:
- Prioritize: sysconfig-a (disk health), sysconfig-r (aggregate/RAID), EMS logs (errors/alerts), all-coredump (panics), platform-sensors (temperature/voltage/fans), any file with 'alert' in name
- Output 3-5 items, each with [✅]/[⚠️]/[❌] + category + finding
- If nothing wrong: "✅ 未发现明显异常"
- Max ~15 tool calls, concise output

## 2. Database (models/db.py + core/database.py)
Add `ai_summary TEXT` column to sessions table with auto-migration.

## 3. Background analysis (api/upload.py)
After parsing completes and status is set to 'done', spawn a background task:
- Create LLMService with POST_UPLOAD_SYSTEM_PROMPT
- Ask: "请分析此 ASUP 日志的健康状况"
- AI uses tools to read key files
- Store result in session_row.ai_summary
- If AI fails, store empty string (don't block upload flow)

## 4. API endpoint (api/upload.py or new route)
Add GET /api/v1/sessions/{session_id}/ai_summary
Returns { session_id, ai_summary }

## 5. Frontend — show on cards
- Card data interfaces: add aiSummary?: string
- TextFileCard / EMSFileCard / XMLFileCard: show 💡 badge next to filename if ai_summary is non-empty
- On hover: tooltip with summary text
- Also show in NodeHUD and ViewerPage session meta

## 6. Flow
Upload → parse → done → background AI analysis → store ai_summary
User opens card → sees 💡 → hover for summary

## Rules
- AI analysis must NOT block upload response
- Failed AI analysis silently stores empty string
- Use existing LLMService class, just with different system prompt
