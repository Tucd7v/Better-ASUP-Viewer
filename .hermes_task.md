Fix MermaidDiagram flashing/delayed rendering in AIChatPanel and Insight panel.

## Problem
Mermaid SVG only renders after sidebar drag stops. During drag, the container width changes constantly, causing ReactMarkdown to regenerate components, and Mermaid renders get aborted or produce wrong-sized SVGs. Result: diagram flickers during resize and only appears stable after drag ends.

## Root cause
Mermaid renders synchronously at component mount time with the container's current width. When the parent panel width changes (sidebar resize), React re-renders the message list, ReactMarkdown recreates the code component, and MermaidDiagram re-mounts with a new chart prop causing re-render.

## Fix in MermaidDiagram.tsx
1. Track container width with a ResizeObserver
2. Debounce width changes (200ms) — only trigger re-render when width stabilizes
3. Use a stable key derived from `chart + debouncedWidth` so React doesn't unmount/remount
4. When container width changes, clear SVG and re-render with new dimensions

Implementation:
```tsx
const containerRef = useRef<HTMLDivElement>(null)
const [containerWidth, setContainerWidth] = useState(0)
const widthTimer = useRef<number | null>(null)

useEffect(() => {
  const el = containerRef.current
  if (!el) return
  const observer = new ResizeObserver((entries) => {
    const w = entries[0].contentRect.width
    if (w === containerWidth || w === 0) return
    if (widthTimer.current) clearTimeout(widthTimer.current)
    widthTimer.current = window.setTimeout(() => {
      setContainerWidth(w)
    }, 200)
  })
  observer.observe(el)
  return () => observer.disconnect()
}, [])

// Use containerWidth in dependency array so diagram re-renders after resize stabilizes
useEffect(() => {
  if (!containerRef.current || !containerWidth) return
  // ... render mermaid
}, [chart, uid, containerWidth])
```

## Also check
- If ReactMarkdown `components` object is not memoized, every parent re-render creates new component references causing remount. In AIChatPanel, wrap `components` in `useMemo`.
- Make sure the `key` prop on messages is stable (existing `i` index in map is fine for chat).
