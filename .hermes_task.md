Add a Duplicate button to XML, TXT, and EMS card headers in the Viewer.

## 1. Card Data interfaces — add onDuplicate
In each of these files, add `onDuplicate: () => void` to the CardData interface:
- TextFileCard.tsx (TextFileCardData)
- EMSFileCard.tsx (EMSFileCardData)
- XMLFileCard.tsx (XMLFileCardData)

## 2. Card header — add [⧉] Duplicate button
In each card's header row (next to collapse/hide buttons), add a small button:
```
[⧉] Duplicate
```
When clicked, calls `data.onDuplicate`.

## 3. ViewerPage — handleDuplicateCard function
Add a `handleDuplicateCard` that:
1. Takes the current node's data
2. Creates a new nodeId (`crypto.randomUUID()`)  
3. Copies all data fields, prepends `[Duplicate]` to filename
4. Adds `__duplicate: true` flag to data
5. Uses `useReactFlow().addNodes()` to spawn the new card at a slightly offset position (node.position.x + 30, y + 30)
6. Passes `onDuplicate` callback to the new node too (so duplicates can be duplicated again)

## 4. AI isolation
The duplicate cards should NOT be exposed to AI. Where the AI panel reads nodes/sessions, skip any node where `data.__duplicate === true`.

## 5. Pass onDuplicate from ViewerPage
Where nodes are rendered (nodeTypes), pass `onDuplicate: () => handleDuplicateCard(node)`.

## Verify
- Click [⧉] on any XML/TXT/EMS card → new card appears with [Duplicate] filename
- New card is independently scrollable/searchable
- AI panel does not see the duplicate
