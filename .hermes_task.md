Fix AISummaryPanel collapse issue in ViewerPage.tsx (around line 181-200).

## Root cause
The useEffect unconditionally adds ALL section labels to openSections whenever `sections` changes. This overwrites the user's manual collapse action. When new AI summary data arrives (polling), sections changes → all sections re-open.

## Fix
Use a `ref` to track section labels from the previous render. Only auto-open sections that are NEW (not previously seen). Don't touch sections the user manually closed.

```tsx
const prevLabelsRef = useRef<Set<string>>(new Set())

useEffect(() => {
  const currentLabels = new Set(sections.map(s => s.label))
  const prevLabels = prevLabelsRef.current
  
  setOpenSections(prev => {
    const next = new Set(prev)
    // Auto-open only NEW sections
    currentLabels.forEach(label => {
      if (!prevLabels.has(label)) next.add(label)
    })
    return next
  })
  
  prevLabelsRef.current = currentLabels
}, [sections])
```

Also remove the initial useState that opens all sections — start with empty set.
