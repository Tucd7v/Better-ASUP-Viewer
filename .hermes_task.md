Improve AI Summary UI and add config toggle.

## 1. Remove 💡 from NodeHUD
In NodeHUD.tsx, remove the 💡 badge from node rows (hud-ai-summary span).

## 2. AI Summary button → clickable dialog
In ViewerPage toolbar: change the 💡 AI Summary span from hover tooltip to a clickable button.
When clicked, open a modal/dialog that:
- Groups summaries by node (hostname or serialNum)
- Each node section: header with node name, collapsible content
- Content rendered as Markdown (use react-markdown with remark-gfm, already installed)
- Close button or click outside to dismiss

## 3. Config toggle in aiconfig.yaml
Add:
```yaml
ai_auto_analysis:
  enabled: true  # set to false to disable AI analysis after upload
```

In `services/llm.py` or `core/config.py`: read this setting. If disabled:
- Backend: skip _start_ai_analysis
- Frontend: AI Summary button shows disabled state with tooltip "AI analysis is disabled"

## 4. Frontend reads config
Add an API endpoint GET /api/v1/config that returns { ai_auto_analysis: { enabled: bool } }
OR pass it through the session status response.
On load, fetch config. If disabled, button is grayed out with tooltip.

## 5. Markdown rendering
Dialog content uses `<ReactMarkdown>` with remarkGfm plugin.
Style the markdown with clean typography.

## Verification
- Upload → AI analyzes → click 💡 button → dialog shows grouped summaries
