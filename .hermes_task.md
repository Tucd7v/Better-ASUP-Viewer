## Issue 2: Preserve card state on swap
Current approach clones nodes `{ ...n }` which causes React to treat them as new objects — losing state. Fix:

In handleSwapCard, DON'T clone — mutate in place and return new array:
```ts
const handleSwapCard = useCallback((sourceId: string, targetId: string) => {
  setNodesForTab(activeTabIdRef.current, (prev) => {
    const next = [...prev]
    const sourceIdx = prev.findIndex(n => n.id === sourceId)
    const targetIdx = prev.findIndex(n => n.id === targetId)
    if (sourceIdx < 0 || targetIdx < 0) return prev
    // Swap array positions so visual order changes
    const tmp = next[sourceIdx]
    next[sourceIdx] = next[targetIdx]
    next[targetIdx] = tmp
    return next
  })
}, [setNodesForTab])
```

This swaps the actual node references (same objects) so React preserves component identity via key.

## Issue 3: Don't break sidebar file-drop-to-replace
The sidebar file-drop uses `handleDrop` which reads fileId from `dataTransfer.getData('text/plain')`. Card-to-card swap also uses the same dataTransfer field. These MUST be distinguished:

- Card-to-card grip drag: set custom data type `card-drag: true`
- Sidebar file-drop: only fileId in text/plain

In SplitCard onDrop: check if `swapDragSource !== null` first (card swap), otherwise delegate to `handleDrop` (file replace). This is already correctly ordered.

ALSO: remove `marginLeft: 16` from SplitCard — NO layout changes to SplitCard itself. The grip lives inside the card component.
