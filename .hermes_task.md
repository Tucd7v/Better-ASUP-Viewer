Sort sessionRows in FileTree so HA paired nodes are displayed consecutively.

Currently nodes are listed in whatever order they arrive from the backend. HA paired nodes (e.g. node01A and node01B) might be separated by other nodes.

Fix: after computing haPairSessionIds, sort sessionRows so that paired nodes are adjacent:
- Walk through sessionRows
- For each node, if it has a partner in the list, ensure the partner appears right after it
- Preserve relative order of non-paired nodes

Simplest approach: create a new sorted array.
```tsx
const sortedSessionRows = useMemo(() => {
  const result: typeof sessionRows = []
  const used = new Set<string>()
  
  for (const session of sessionRows) {
    if (used.has(session.sessionId)) continue
    result.push(session)
    used.add(session.sessionId)
    
    // Find partner and add next
    const partnerName = normalizeHostname(session.partnerHostname)
    if (partnerName) {
      const partner = sessionRows.find(s => 
        normalizeHostname(s.hostname) === partnerName && !used.has(s.sessionId)
      )
      if (partner) {
        result.push(partner)
        used.add(partner.sessionId)
      }
    }
  }
  return result
}, [sessionRows, haPairSessionIds])
```

Then use `sortedSessionRows` instead of `sessionRows` in the render map.
