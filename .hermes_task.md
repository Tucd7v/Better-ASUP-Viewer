Fix three drag-to-reorder issues in Grid mode (SplitGrid, ViewerPage.tsx):

## Issue 1: XML column resize conflicts with card drag
The 28px full-width drag overlay covers XML table header `<th>` elements, so dragging a column resize handle also triggers card reorder.

## Issue 2: Card header buttons blocked
The overlay covers [⧉]/[-]/[×] buttons in card headers, making them unclickable.

## Issue 3: Card state lost after swap
When swapping nodes in the array, React unmounts/remounts card components, losing internal state (scroll position, highlight, search matches, etc.).

## Fix for Issues 1+2: Narrow grip handle
In SplitCard, replace the 28px full-width drag overlay with a small grip handle on the left edge:
- Remove the full-width overlay div
- Add a 16px-wide, 28px-tall grip handle at absolute position left:0, top:0
- Grip handle shows `⋮⋮` in gray, cursor: grab
- Only the grip is draggable (not the rest of the card)
- Card content gets margin-left: 16px to make room

## Fix for Issue 3: Swap positions instead of nodes
In ViewerInner.handleSwapCard, instead of swapping array indices, swap only the `position` (x, y) of the two nodes. This preserves React component identity, keeping card internal state.

```ts
const handleSwapCard = useCallback((sourceIdx: number, targetIdx: number) => {
  setNodesForTab(activeTabIdRef.current, (prev) => {
    const next = prev.map(n => ({ ...n }))
    const tmp = next[sourceIdx].position
    next[sourceIdx] = { ...next[sourceIdx], position: next[targetIdx].position }
    next[targetIdx] = { ...next[targetIdx], position: tmp }
    return next
  })
}, [setNodesForTab])
```

This way ReactFlow sees the same nodes with new positions — no unmount/remount.
