Fix XML card horizontal scrollbar: when card width >= ~931px, the scrollbar disappears because `needsScroll` compares tableWidth to container width, but individual cells still overflow.

Root cause at line 281:
```ts
const needsScroll = tableWidth > 0 && tableWidth > width - 20
```

When card is wide, `tableWidth` may be less than `width - 20`, so `overflowX: hidden` hides content. But cells with `white-space: nowrap` and `overflow: hidden` still clip content.

Fix: replace `overflowX: needsScroll ? 'scroll' : 'hidden'` with `overflowX: 'auto'`. Let the browser handle scrollbar visibility natively — it correctly detects when content overflows the container, regardless of card width.

Also keep the existing `overflowY: 'auto'` for vertical scroll.
